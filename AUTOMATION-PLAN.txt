AQRpool / Pooledit - แผนทำระบบให้เป็น "ฟูลออโต้"
============================================================
อัปเดต: 2026-06-30


[1] ระบบที่เป็นออโต้อยู่แล้ว (ไม่ต้องแตะ)
------------------------------------------------------------
- Backup รายวัน           : ตี 2 อัตโนมัติ (lib/scheduler.ts)
- เช็คอิน + หักโควต้า       : หักแพ็กเกจอัตโนมัติตอนสแกน
- คูปอง                    : เช็ควันหมดอายุตอนใช้งานเอง
- ตรวจสลิป OCR            : auto-approve ถ้าสลิปตรง + เปิด setting (ออโต้ครึ่งทาง)


[2] ระบบที่ทำให้ "ฟูลออโต้" ได้
------------------------------------------------------------

ระบบ            | ตอนนี้ทำมือ                  | ทำออโต้เป็น
----------------|------------------------------|--------------------------------------------------
แพ็กเกจ         | ไม่มีแจ้งเตือนหมดอายุ          | มาร์ค "หมดอายุ" อัตโนมัติเมื่อถึงวัน
                |                              | + เตือนล่วงหน้า 7/3/1 วัน
                |                              | + เตือนต่ออายุ
----------------|------------------------------|--------------------------------------------------
การจอง          | admin กดยืนยันเอง            | auto-confirm หลัง X ชม.
(Booking)       |                              | + auto-cancel เมื่อเลยเวลา (no-show)
                |                              | + เตือน 24 ชม. / 1 ชม. ก่อนจอง
----------------|------------------------------|--------------------------------------------------
เติมเงิน        | admin กด approve / reject     | auto-reject สลิปซ้ำ / อ่านไม่ออก
(Topup)         |                              | หลัง 24-48 ชม.
----------------|------------------------------|--------------------------------------------------
ออเดอร์ร้านค้า   | admin กดยืนยันจ่าย / จัดส่ง    | auto-verify สลิป (ใช้ OCR เดิม)
                |                              | + auto-cancel ออเดอร์ที่ไม่จ่ายใน 7 วัน
                |                              | + แจ้งเตือนเมื่อส่ง
----------------|------------------------------|--------------------------------------------------
ลางาน          | admin อนุมัติเอง             | auto-reject วันที่ชน
(Leave)         |                              | + auto-approve ลาป่วย < 1 วัน
                |                              | + ส่งอีเมลแจ้งผล
----------------|------------------------------|--------------------------------------------------
แจ้งเตือน        | ดึงข้อมูลแบบ polling          | ส่ง push / อีเมลอัตโนมัติเมื่อมี event
                |                              | + สรุปรายวันให้ admin
----------------|------------------------------|--------------------------------------------------


[3] ลำดับแนะนำ (คุ้มสุด / ROI สูง)
------------------------------------------------------------
1. แพ็กเกจหมดอายุ + เตือนต่ออายุ   -> กระทบสมาชิกทุกคน ไม่มีอะไรบล็อก
2. Booking auto-confirm + no-show  -> ลดงาน admin ~70%
   + เตือนก่อนจอง
3. ออเดอร์ auto-verify จ่าย / ส่ง   -> ใช้ OCR ที่มีอยู่แล้ว


[4] โครงสร้างที่พร้อมใช้แล้ว
------------------------------------------------------------
- lib/scheduler.ts : รัน job ตามเวลา (setInterval + marker กันรันซ้ำ)
- lib/mailer.ts    : ส่งอีเมล (Brevo SMTP) ใช้ส่ง OTP อยู่แล้ว
- slipVerify.ts    : OCR ตรวจสลิป นำมาใช้ซ้ำกับออเดอร์ได้
- db.transaction() : ล็อกกันชนกันตอนหักโควต้า
